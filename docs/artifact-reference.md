# Artifact Reference

This reference explains the main files generated by a PermitGraph scan.

## Run Directory

Scan artifacts live under:

```text
.agent-permit/runs/<run_id>/
```

Generated artifacts are local evidence. Treat them as sensitive until reviewed.

## Core Artifacts

| File | Audience | Purpose |
| --- | --- | --- |
| `summary.md` | reviewer | short human-readable scan result |
| `risk-report.md` | reviewer | expanded risk and remediation narrative |
| `permit.yaml` | tools, reviewer | deterministic permit status |
| `raw-findings.json` | tools, AI agents | normalized finding records |
| `graph-paths.json` | tools, AI agents | source-to-sink access paths |
| `controls.json` | tools, reviewer | controls checked before approval |
| `run-metrics.json` | tools, operators | timing, counts, usage metadata |
| `agent-investigation.md` | reviewer | optional Deep Agent explanation |
| `openrouter-usage.json` | operators | optional model usage and cache telemetry |

## Review Order

1. Start with `summary.md`.
2. Confirm status in `permit.yaml`.
3. Inspect `raw-findings.json` for rule and evidence records.
4. Inspect `graph-paths.json` for source-to-sink paths.
5. Inspect `agent-investigation.md` only as cited explanation.

## Permit Status

`approved` means no configured agent-access risk matched.

`needs_review` means a reviewer must inspect evidence before unattended access continues.

`blocked` means unattended access should stop until remediation or explicit exception.

## Sensitive Data Rules

- Do not commit `.agent-permit/runs/`.
- Do not publish generated artifacts without redaction review.
- Do not expose raw secret values.
- Proof packs are safer than raw run folders, but still require review.
