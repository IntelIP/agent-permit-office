# Code Of Conduct

Agent Permit Office is a security project. Treat reports, mistakes, and disagreements with care.

Expected behavior:

- be direct and respectful
- assume security reports may contain sensitive context
- keep discussions technical and evidence-based
- redact secrets, customer data, and private repository details
- help contributors reproduce scanner behavior with minimal examples

Unacceptable behavior:

- harassment, threats, or personal attacks
- publishing private vulnerabilities before maintainers can respond
- posting real credentials or private customer data
- using issues or discussions to pressure disclosure of sensitive information
- repeated disruption after maintainers ask for a behavior change

Maintainers may edit, hide, lock, or remove comments and may limit participation when needed to keep the project safe and useful.

Before public launch, enforcement is handled by the project maintainer. After public launch, this file should be updated with a named reporting channel.
